# 詳細設計: 投資信託 運用レポート ジェネレーター

`PLAN.md` を唯一の仕様源とする詳細設計。PDF・`mupdf`・`defs.json` は対象外
（旧設計の名残で削除済み）。

## データ取得（`ids.json` の ISIN ごと）

- `https://www.rakuten-sec.co.jp/web/fund/detail/?ID=<ID>` を Node 18+ 組み込みの
  `fetch` で取得。ページは UTF-8、ヘッダ不要のベア GET。
- **基準価額系列**（ページ内 Highstock の `data:[[epoch_ms, value], …]`）:
  `基準価額` / `基準価額+分配金` を抽出。そのファンドのページに**存在する系列のみ**
  出力（1〜2 本）。両系列は円単位で同一スケール、同一タイムスタンプで日次整列。
  `分類平均`（基準が異なり円換算で NAV と比較不能）はチャートから除外。
- **純資産系列**（同じく Highstock の `純資産`、単位は億円）: 存在すれば抽出。
  NAV と桁が異なるため**第 2 軸**に乗せる（`fund.netAssets`、必須ではない）。
- **統計テーブル**: ページの `パフォーマンス` HTML テーブルから
  `リターン(年率）` / `リスク(年率）` / `シャープレシオ（ＳＲ）` × (6ヵ月/1年/3年/5年)
  を**表示文字列のまま**スクレイプ。**計算は一切しない**。
- **ファンド名**・**基準日**（最新の基準価額データ日）もページから取得。

## デッキ（`template.pptx` = 3 スライド → データ 6 スライド）

1. **タイトル** — `{fund_name}`, `{report_date}`（基準日, `YYYY年M月D日`）。
2. **チャート**（1 枚を 4 回繰り返し使用）— `{chart_title}`（"基準価額（6ヶ月）"…）、
   チャートは alt text `nav_chart` でバインド。**2 軸**構成で**凡例は下部**に表示。
   - 主軸（左, `axis1`）: NAV を生の円値でプロット（リベースしない）、折れ線。
   - 第 2 軸（右, `axis2`）: 純資産（億円）を**面グラフ**で表示。最小値 0、塗りは
     `accent6`（系列上書きで塗りが失われるため `data.json` 側で明示）、NAV 折れ線に
     隠れないよう**最背面**に描画。純資産が無いファンドでは単軸にフォールバック。
   - 日付軸は `date_labels`（epoch→Excel シリアル）+ `date_labels_format: "yyyy/mm"`。
     主目盛は期間ごと: 6ヶ月→1ヶ月 / 1年→1〜2ヶ月 / 3年→6ヶ月 / 5年→1年。
   - **サンプリング**: 6ヶ月・1年は日次、3年・5年は月末（各月最終営業日）。
   - `基準価額+分配金` が `基準価額` と一致しても両方残す（存在すれば出力。分配金あり
     ファンドでは分岐する）。
3. **テーブル** — alt text `stats_table` でバインド。ヘッダ含む 4×5 グリッド全体を
   スクレイプ行から供給。数値セルはマイナスを reddish（テーマ `accent4`）、プラスを
   dark gray（テーマ `dk1`）で着色（テーマ定義色のみ使用）。ヘッダ・ラベル列は既定色。

## パイプライン（`node reports.js`）

- 既定は `ids.json` の全件。`--id=<ID>` 指定時はそのファンドだけ処理する（開発・確認用）。
- ファンドを**逐次**処理（`PLAN.md` の「順番に」、かつ SlidePack の
  「アカウントあたり未レンダーセッションは 1 つ」制約による）。
- ファンドごと: スクレイプ → `data.json` 生成 → `template.pptx` と `zip`（シェルの
  `zip`、一時ファイル）→ API 4 ステップ（毎回新規セッション、`render_succeeded` を確認）
  → ダウンロード → `out/{ID}-YYYY-MM-DD.pptx`（基準日）。
- **エラー継続**: 失敗は ISIN と理由をログ出力し、成功した PPTX は残す。
  1 件でも失敗したら非ゼロ終了。
- API キーは `process.env.SLIDEPACK_API_KEY`（mise 経由）。

## ビルド成果物

- `template.pptx` は `pre-template.pptx` から使い捨て（**コミットしない**）スクリプトで
  生成し、`inspect-template.sh` で検証。コミットするのは `template.pptx` のみ。
  `pre-template.pptx` を変更した場合は都度セッションで個別対応する。
- npm 依存なし（組み込みの `fetch`/`FormData`/`Blob` + シェルの `zip`）。

## モジュール構成（目安）

`reports.js`（オーケストレータ）/ `scrape.js`（取得・解析）/
`transform.js`（→ `data.json`）/ `slidepack.js`（API 4 ステップ）。
開発中は `data.json` をスキルの `validate.py` で検証する。
